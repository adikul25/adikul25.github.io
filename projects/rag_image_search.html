Image RAG Search System

  • Tech Stack: Python, FastAPI, Streamlit, FAISS, Llama 3.2-11B Vision, Docker
  • Github URL: Project Link

Overview
Developed an Image RAG Search System leveraging Llama 3.2-11B Vision to generate intelligent image descriptions, which are stored alongside paths in a FAISS vector database for efficient retrieval based on natural language user queries. This enables semantic image search using plain language instead of traditional keyword-based approaches.

Key Features
• AI-powered image description generation using Llama 3.2 Vision model
• FAISS vector database for efficient semantic similarity search
• Natural language query interface for intuitive image retrieval
• FastAPI backend for scalable REST API endpoints
• Streamlit frontend for user-friendly interface
• Docker containerization for easy deployment
• Asynchronous processing to handle Llama's 30 RPM rate limits

Architecture
The system follows a three-stage workflow: (1) Images are base64-encoded and processed through Llama 3.2 Vision to generate detailed descriptions, (2) Descriptions are converted to embeddings and indexed in FAISS, (3) User queries are embedded and matched against the index for semantic retrieval. The FastAPI backend handles async processing for smooth image description generation and retrieval, ensuring efficient image transfer and search.

Use Cases
Ideal for photo libraries, digital asset management, e-commerce product catalogs, and any application requiring intelligent image search beyond traditional metadata-based approaches.
